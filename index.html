A Websockets Project

Uno, was a fun class project during my Junior year. The game was my own idea to build, with the requirement to make a websocket application. The focus wasn't as much on the UI, but the constant use of websockets. Wether a player was to send a message to an opponent, draw a card, or play a card, websockets would process the action and display it to the other user in a variety of ways.

Zion Wildflower App

Progressive Web Application

Zion Wildflower is an app that is being produced to the public. It is projected to be used by thousands of people visiting Zion National Park. The goal was to make a PWA instead of the native app that was already produced because it was only being used once per client, then deleted. We redesigned the entire app for a better user experience as a whole. It could now be used as a web app or even installed like a native web app, but using PWA capabilities (including the important need for use offline).

Zion Dashboard

Stat Based App for Zion National Park

Zions Park Data Dashboard is a project that has been replicated for multiple National Parks in the United States. Each of these parks were keeping track of these stats manually with paper and pencil. They now have machine learning cameras and radars that help them keep track of visitor stats. I developed the web app to display the API data in multiple formats for the National Parks employees to use.

Zion VEM PWA

Counting App for National Parks

Zion VEM is an extremely important app for the National Parks. They hire multiple interns to use clickers, pencils, and paper to keep track of visitor counts and current conditions of each trail in the park. This app was created for these interns to use their mobile phones for a much more accurate and efficient way of tracking these stats. Because of the need to use the app on trails without an internet connection, we made the app progressive for efficient offline use.
